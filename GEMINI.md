# Reglas Generales de Proyecto

## 1. 🛑 Cero Código sin Autorización (Regla de Oro Global)
- **ESTRICTAMENTE PROHIBIDO** escribir, modificar, eliminar o refactorizar código sin mi autorización explícita previa.
- Siempre debes detenerte después de proponer una solución y esperar mi confirmación (ej. "Procede", "Aprobado") antes de ejecutar cambios.

## 2. 🛡️ Control de Versiones Estricto (Git)
- **PROHIBIDO** realizar `git commit`, `git push` o mezclar ramas sin mi orden explícita.
- Todo el código debe ser probado localmente antes de integrarlo.
- Mensajes de commit y nombres de ramas siempre en Español.

## 3. 🧠 Visión General y Pensamiento Crítico (Big Picture)
- No saltes directamente a solucionar el síntoma del problema. Primero analiza la imagen general y la arquitectura del proyecto.
- Presentar siempre: Diagnóstico, Enfoques posibles (Pros/Contras) e Impacto.

## 4. ⚖️ Integridad de la Lógica y Frontend
- No simplificar componentes. Itera sobre el código pre-existente.
- Respetar la lógica de negocio y estilos establecidos.

## 5. ⚡ Optimización de Tokens y Flujo de Trabajo
- Sé directo, conciso y técnico.
1. Análisis -> 2. Propuesta (Pros/Contras) -> 3. Pausa -> 4. Ejecución -> 5. Verificación/Git.
